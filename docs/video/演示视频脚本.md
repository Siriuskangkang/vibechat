# VibeChat · 完整功能演示视频脚本

> 目标时长：**3–6 分钟**（本脚本目标 ~4:40，留剪辑余量）
> 用途：选手自录；可**配音或字幕**说明。本脚本含**完整旁白文案**，可直接用于 AI 配音（TTS）。
> 赛题展示点全覆盖：①情绪输入 ②AI 情绪识别 ③自动匹配 ④匿名聊天 ⑤线上地址访问 ＋ 技术亮点（双接口 / 情绪场 / 降级 / 安全）。
> 依据：`docs/第三期 VibeChat.md` 提交清单第 2 项；分镜对齐 `specs/.../09-测试验收与演示.md` §4。

---

> 🔗 **录制操作权威**：逐动作步骤、文件命名、分屏投屏设置见 [`演示视频录制清单.md`](./演示视频录制清单.md)（16 镜）。本文件为其提供**旁白 / 配音文案**。

## 0. 录制前准备（避免翻车）

- [ ] **线上烟测**：`https://vibechat.wukangkang.com` 比赛前夜打通一次全链路（输入→匹配→进房→收发→离场）。
- [ ] **mock 多用户**：开 2–3 个无痕窗口（不同 `sid` = 不同匿名身份）模拟多人同房；或用 `scripts/ws_test.py`。
- [ ] **双接口就绪**：`.env` 里 `LLM_PROVIDER` 能在 `openai` / `anthropic` 间切换，两路 key 都配好。
- [ ] **录屏设置**：1920×1080，浏览器缩放 100%，关闭通知/书签栏，鼠标平滑。
- [ ] **配音/字幕**：见 §4（可用 fal-audio TTS + ffmpeg 字幕，或直接用已接入的阿里云百炼 CosyVoice）。
- [ ] 数值以**实际页面输出为准**（脚本中的 -0.7 / 0.8 / 87% 等为示例）。

## 2. 分镜详情（画面操作 / 完整旁白 / 字幕 / 要点）

### 镜 1 · 开场（0:00–0:15）
- **画面**：首页，星尘缓慢下落，标题渐显。
- **旁白**：「VibeChat——一款 AI 驱动的情绪社交应用。它只做一件事：让当下情绪，决定你和谁相遇。」
- **字幕**：VibeChat · 让当下情绪决定相遇
- **要点**：1 句话说清定位。

### 镜 2 · 线上访问（0:15–0:25）
- **画面**：地址栏输入 `vibechat.wukangkang.com` 回车，页面加载。
- **旁白**：「已经部署在公网，评委现在就可以打开这个地址体验。」
- **字幕**：已部署公网 · 线上可访问
- **要点**：对应⑤，规避「无线上地址」严重扣分项。

### 镜 3 · 情绪输入（0:25–0:35）
- **画面**：首页输入框，输入「明天要汇报，脑子停不下来，睡不着」，点击分析。
- **旁白**：「先说一句此刻的心情。」
- **字幕**：① 情绪输入
- **要点**：对应①。

### 镜 4 · AI 情绪识别（0:35–0:55）
- **画面**：结果卡出现——主情绪「焦虑」、valence −0.7、arousal 0.8、intensity、关键词、一句话解读。
- **旁白**：「AI 把这段心情拆解成结构化的情绪：主情绪是焦虑，偏负面、激动度很高。它读懂的不只是一个词，而是情绪的色彩、强度，还有倾诉的倾向。」
- **字幕**：② LLM 结构化情绪分析（向量 + 标签 + 解读）
- **要点**：对应②；强调 LLM 真正产出结构化向量（非贴标签）。

### 镜 5 · 情绪可视化（0:55–1:15）
- **画面**：`EmotionColorWheel` 涟漪+光晕落点落在「消极·激动」象限，颜色冷紫。
- **旁白**：「这颗发光的落点，就是你的情绪在二维情绪空间里的位置——消极、紧绷。」
- **字幕**：情绪可视化 · 落点定位
- **要点**：开放发挥「情绪可视化」。

### 镜 6 · 自动匹配（1:15–1:40）
- **画面**：从落点连线到「深夜焦虑」房，显示「87% 同频」，下方列出备选房。
- **旁白**：「系统用向量相似度，把你匹配到此刻和你最同频的房间——87%。匹配不是随机的，是情绪向量真实算出来的，下方还能看到其他候选房。」
- **字幕**：③ 向量相似度匹配 · 情绪真实决定进哪间房
- **要点**：对应③；规避「情绪与匹配脱节」严重扣分项——**重点镜头**。

### 镜 7 · 进房·匿名身份（1:40–1:55）
- **画面**：进入房间，顶部出现匿名昵称（如「失眠的云」）+ 几何情绪色头像。
- **旁白**：「进房后，系统为你生成一个匿名的名字和头像——不暴露真实身份。」
- **字幕**：匿名身份系统生成
- **要点**：对应必做第 3 条「身份由系统生成」。

### 镜 8 · AI 主持破冰（1:55–2:10）
- **画面**：居中的「房间主持」气泡弹出一句开场白。
- **旁白**：「房间里的 AI 主持会先开口，说一句贴合氛围的开场白，帮你破冰。」
- **字幕**：AI 主持 · 自动破冰
- **要点**：开放发挥「对话辅助」。

### 镜 9 · 分屏双端·同频相遇 + 跨端聊天（2:10–2:45）★★★核心
- **画面**：左 PC 浏览器 / 右手机投屏。两端输入**相同的话** → 都匹配进**同一间房** → 两人在房里实时聊几句（左发右收、右发左收）。
- **旁白**：「换个场景：两个不同设备的人，写下同一句话。AI 读出相同的情绪，把他们送进同一间房，然后真的聊了起来——一端发，另一端实时收到。这正是 VibeChat 的核心：被同一种心情抓住的人，值得在一间房里相遇。」
- **字幕**：相同情绪 → 同频匹配 → 同房实时对话
- **要点**：匹配 + 收发双核心；规避「情绪与匹配脱节」「聊天无法收发」两个严重扣分项——**最重要镜头**。操作详见《演示视频录制清单.md》镜 9–10（分屏投屏设置见其 §3）。

### 镜 10 · 会呼吸的情绪房间（2:40–3:05）★
- **画面**：随着不同情绪的人进房/发言，房间背景色调流动变化，顶部「共鸣度」数值随之上扬。
- **旁白**：「这是 VibeChat 最特别的地方——房间会呼吸。它的色调、共鸣度，跟着此刻同处一室的人的情绪实时流动。情绪不再只是匹配时贴一次的标签，而是一个流动的、能被看见的场。」
- **字幕**：★ 会呼吸的情绪房间 · 情绪实时流动
- **要点**：产品最大差异化（spec 10）；演示「哇」时刻——**重点镜头**。

### 镜 11 · 离场总结·情绪旅程（3:05–3:30）
- **画面**：点「离开房间」→ 加载 → 离场页：AI 情绪小结 + 一句 takeaway + 情绪旅程弧线（你来时→此刻）。
- **旁白**：「离开时，AI 回看你们的对话，给出一段温柔的情绪小结，并画出你从进房到此刻的情绪轨迹——你来时紧绷，此刻似乎轻了一点。」
- **字幕**：离场情绪小结 + 旅程轨迹
- **要点**：开放发挥「记忆与回顾」。

### 镜 12 · 双接口切换（3:30–4:05）
- **画面**：改 `.env` 的 `LLM_PROVIDER=openai` → 重启 → 跑一次分析；再切 `anthropic` → 重启 → 同一输入再跑一次。两次都正常输出结果。
- **旁白**：「后端适配了两套标准接口——OpenAI 标准和 Anthropic 标准。切换一个环境变量，同一句输入，两套接口都能跑通。」
- **字幕**：双标准接口 · 环境变量切换（OpenAI / Anthropic）
- **要点**：技术核心；规避「只支持一种」严重扣分项——**重点镜头**。

### 镜 13 · 异常降级（4:05–4:25）
- **画面**：模拟 LLM 异常（断 key / 改错地址）→ 再次输入 → 仍返回结果（规则推断模式，标注「离线模式」），匹配照常。
- **旁白」：「即使 LLM 接口出问题，系统也会降级到规则推断，核心流程不中断——保证演示永远不崩。」
- **字幕**：异常自动降级 · 链路不中断
- **要点**：对应「稳定演示」。

### 镜 14 · 安全边界（4:25–4:40，可选）
- **画面**：输入极端倾向文字 → 进入专属守护房，顶部置顶求助资源卡。
- **旁白」：「当识别到极端情绪，产品会温柔地把你引到一个安全的房间，并给出 24 小时心理援助热线——情绪社交，也要有边界。」
- **字幕」：安全边界 · 极端情绪求助
- **要点」：开放发挥「安全边界」。

### 镜 15 · 收尾（4:40–4:50）
- **画面」：回首页，星尘背景，地址浮现。

## 3. 录制与剪辑要点

- **节奏**：重点镜头（⑥匹配 / ⑨收发 / ⑩情绪场 / ⑫双接口）放慢、给足停留；过渡镜头可加速。
- **避免冷场」：⑨群聊前先把 mock 窗口准备好，录制时轮流切窗口发言；或用 `DEMO_BOT_TENANT=on` 让 AI 房客陪聊。
- **真实数据」：旁白里的数值录制时以页面实际输出替换，保持口型/字幕一致。
- **剪辑」：总长压在 3–6 分钟；片头 2s 静帧带标题，片尾留地址 3s。
- **配音/字幕」：见 §4。

## 4. 配音与字幕制作（AI 流水线）

> 你只需录好画面，配音和字幕都能用 AI 自动生成。两条路线：

**路线 A · 用开源 skill（开放生态）**
- **AI 配音（TTS）**：`fal-ai-community/skills@fal-audio` —— 文案转语音。
  - 安装：`npx skills add fal-ai-community/skills@fal-audio -g -y`
- **自动字幕」：先用 `inference-sh/skills@speech-to-text`（语音转带时间戳文字）生成 SRT，再用 `josiahsiegel/claude-plugin-marketplace@ffmpeg-captions-subtitles`（ffmpeg 烧录字幕到视频）。
  - 安装：`npx skills add inference-sh/skills@speech-to-text -g -y` ／ `npx skills add josiahsiegel/claude-plugin-marketplace@ffmpeg-captions-subtitles -g -y`

**路线 B · 用已接入的阿里云百炼（推荐，零新依赖）**
- **配音」：百炼 CosyVoice（中文 TTS，质量高）—— 本脚本 §2 每镜「旁白」可直接作为输入文本。
- **字幕」：百炼语音识别（或 OpenAI Whisper）从配音音频生成带时间戳 SRT → ffmpeg 烧录。
- 优势：项目已接百炼，复用现有 key 与 SDK，无需额外注册。
