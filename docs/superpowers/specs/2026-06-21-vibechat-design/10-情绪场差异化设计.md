# 10 · 情绪场差异化设计（★ 产品最大特点）

> 本文档定义 VibeChat 在"题目要求之外"的核心竞争力。vibecoding 时代功能完整不再是壁垒，**壁垒是产品立意的高度 + 一个别人来不及抄的体验锚点**。

---

## 1. 立意：情绪是流动的场，不是匹配的标签

大多数选手会把"情绪"做成**匹配时贴一次的标签**，匹配完进房就退化为普通聊天室——正是赛题警告的"普通聊天室加 AI 标签"。

VibeChat 的差异化立意：**把情绪升级为贯穿房间的、流动的、可感知的连续维度**。一句话记忆点：

> **「会呼吸的情绪房间」** —— 房间的颜色、氛围、共鸣度，随此刻同处一室的人的情绪实时流动。

这把"多人 + 实时 + 情绪 + AI"四者缝合在一个可被评委一眼看见的视觉上。

---

## 2. 三个增强联动（情绪作为连续维度）

```
进房 ───────────────► 对话 ───────────────► 离场
① 房间集体情绪场       ② 情绪共鸣度           ③ 离场情绪旅程
（房间此刻是什么色调）  （大家此刻多同频）      （你来时→此刻的情绪轨迹）
```

三者共用一套底层数据（成员情绪向量），互相支撑，形成"情绪流动的场"的完整闭环。

---

## 3. 共享底座：在线成员情绪聚合（①② 共用，纯计算）

两个增强都建立在"在线成员情绪向量集合"的一次聚合上，**无需额外 LLM 调用**：

```
在线成员向量集 V = {v1, v2, ...}   # 每个成员进房时的情绪向量（来自情绪分析，已有）

房间集体情绪 room_mood = intensity 加权均值(V)        # 房间此刻的整体色调
情绪共鸣度 resonance   = 1 − normalize( mean( dist(vi, room_mood) ) )   # 一致度 ∈[0,1]
```

- `room_mood`：房间此刻的"集体情绪向量"，驱动 ① 色场。
- `resonance`：成员情绪的趋同程度，驱动 ② 共鸣度；趋同→高，分化→低。
- **触发重算**：成员进/离房时。MVP **发言不更新个人向量**（保持轻量，避免每条消息计算/调 LLM）。
- **推送**：随 WebSocket 广播 `{"type":"room_mood","vector":[...],"resonance":0.82,"online":3}`（见 [04 §5.2](./04-系统架构.md)）。

---

## 4. ① 房间集体情绪场（Room Mood Field）

- **前端**：房间背景色 = `room_mood` 映射的颜色（valence→色相，arousal→明度/饱和度）+ Canvas/CSS 缓动粒子。
- **流动感**：成员进出、共鸣变化 → 色场颜色与粒子流向缓动过渡，房间"在呼吸"。
- **成本**：聚合是简单加权均值（已有数据），WS 已有，前端色场是纯 CSS/Canvas。**极轻**。

---

## 5. ② 情绪共鸣度（Resonance）

- **展示**：房间顶部常驻 `此刻共鸣度 82% ↑`（与上一刻对比的升降箭头）。
- **正反馈**：共鸣升高 → 色场变暖 / 粒子向中心聚集；降低 → 色场趋于冷静。
- **意义**：让"情绪驱动社交"在**对话过程中持续可见**——彻底区别于"匹配完就变普通聊天室"。
- **成本**：与 ① 共享同一次聚合，几乎免费；不每条消息调 LLM。

---

## 6. ③ 离场情绪旅程（Mood Journey）

- **复用离场总结的 LLM 调用**：离场总结本来就要调一次 LLM（见 [07 §1.3](./07-AI主持-匿名氛围-安全.md)），让它**多输出一个字段** `mood_end: {valence, arousal}`（用户此刻的情绪）。
  - 输出 schema：`{summary, takeaway, mood_end}` —— **一次调用拿三个值，零额外成本**。
- **旅程可视化**：进房情绪向量（`start`，来自情绪分析）→ `mood_end`（`end`），在**情绪色盘**上画一段轨迹弧线。
- **文案**：「你来时是 {start_primary}，此刻似乎 {轻了一点/沉淀下来/…}」。
- **意义**：情绪闭环的最后一环，承接巡梦"被接住"理念，记忆感强；把"情绪会变化"这件事可视出来。

---

## 7. 轻量化论证（关键：这三个都不贵）

| 增强 | 计算来源 | 额外 LLM 调用 | 新依赖 |
|---|---|---|---|
| ① 房间情绪场 | 成员向量聚合（已有数据） | 无 | 无（纯 CSS/Canvas） |
| ② 情绪共鸣度 | 与 ① 同一次聚合 | 无 | 无 |
| ③ 离场旅程 | 复用离场总结调用，多一字段 | 无 | 无 |

→ **不新增依赖、不每条消息调 LLM、不增加数据库表**，完全符合 [README §3.5 轻量化红线](./README.md)。

---

## 8. 优先级与排期

- 归类为 **P0+（差异化冲刺）**：在 MVP 7 项（保命分）跑通后**立即做**，先于 P1 余量项。
- 实现顺序：先 ①（视觉锚点最重）→ ②（与 ① 同源，顺带）→ ③（复用离场总结）。
- 兜底：聚合/旅程任一异常 → 房间退回固定主题色场、离场只显示文字总结（不影响主流程）。

---

## 9. 对应赛题开放发挥（加分项命中）

- **情绪可视化**：房间情绪场 + 共鸣度 + 离场旅程，三层可视化。
- **产品气质**：让界面与交互真正服务于"情绪社交"主题（赛题重点看项）。
- **记忆与回顾**：离场旅程即情绪轨迹的即时版。

---

## 10. 可测试性

- `room_mood` 加权均值、`resonance` 一致度计算为**纯函数**，单测覆盖（空集、单成员、高分化、高趋同）。
- 离场总结输出含 `mood_end` 字段解析、色盘轨迹点计算为纯函数，单测覆盖。
