# 02 · 产品设计与核心流程

---

## 1. 目标用户画像

| 画像 | 典型诉求 | VibeChat 如何满足 |
|---|---|---|
| **深夜情绪党** | 深夜emo/焦虑，想找人说说但不想被认识的人看到 | 匿名 + 情绪房间，被同频的人接住，无社交压力 |
| **情绪倾诉者** | 想被理解当下心情，而非被"解决问题" | AI 情绪解读 + 同频房间共鸣，情绪被看见 |
| **即时陪伴 seeker** | 此刻就想有人聊，不想填资料、加好友 | 输入一句话即匹配进房，零门槛、即时 |
| **自我觉察者** | 想回看自己一段时间的情绪变化 | 情绪轨迹 + 离场总结，留痕可回顾 |

---

## 2. 核心社交形态：情绪房间·多人同频

- **不是一对一私聊，也不是无主题大厅**，而是按情绪主题划分的**匿名房间**。
- 同一时刻情绪相近的人，被聚进同一间房，**群聊共鸣**。
- 每间房有固定情绪主题（深夜焦虑 / emo 时刻 / 开心想分享 / 愤怒吐槽 / 平静独处 / 迷茫求建议），有专属色场与氛围。
- 房间常驻 AI 主持 + 兜底 bot：**没人也不空，有人即融入**。

### 为什么是房间（而非一对一）

| 维度 | 情绪房间（选定） | 一对一私聊 |
|---|---|---|
| 演示稳定性 | 房间可常驻 bot，演示永不冷场 | 没真人就卡死 |
| 情绪可视化 | 房间=情绪色场，天然载体 | 弱 |
| 实时工程亮点 | WebSocket 多人广播 | 双人通道，亮点弱 |
| 社交压力 | 低（群聊，可只看不说） | 高（一对一要回应） |
| 同频共鸣感 | 强（一群同情绪的人） | 依赖单次匹配质量 |

## 3. 端到端核心流程

```
[情绪输入页]
  用户写一段此刻心情/想说的话 ──► 前端校验(非空/长度上限)
        │  POST /api/emotion/analyze
        ▼
[后端 情绪分析编排]
  1) LLM 单次结构化调用 → {primary, secondary, valence, arousal,
                          intensity, social, keywords, reading}
  2) 落库 emotion_analyses（留痕：证明情绪真被分析）
        │  返回情绪结果
        ▼
[匹配 + 可视化页]
  3) 情绪向量 vs 6 间房锚定向量 → cosine 相似度排序
  4) 选最近房 → 展示"你与『深夜焦虑』房 87% 同频"
  5) 情绪可视化：色盘落点(发光点) + 雷达图 + 房间色块连线
        │  用户点"进入房间"
        ▼
[匿名房间（WebSocket）]
  6) 系统生成匿名身份（昵称 + 情绪色几何头像）
  7) WS 接入房间 → 广播"XX 加入了房间"
  8) AI 主持破冰：据全员情绪生成开场白/话题
  9) 匿名群聊：收发消息、时间序、在线状态、房间色场氛围
        │  过程中：
        │   - 冷场(消息间隔>阈值) → AI 温和抛话题
        │   - 极端情绪/自伤关键词 → 求助卡片
        │   - 兜底：无人/冷场 → bot 匿名发言
        ▼
[离场]
  10) AI 据本次房间对话 → 情绪小总结 {summary, takeaway}
  11) 落库 emotion_timeline（记忆回顾）
  12) 返回情绪轨迹页 / 可再进房
```

**关键不变量**：进哪间房、展示的同频度，**必须可追溯到本次情绪分析的向量输出**（对应扣分项③"情绪与匹配脱节"的反向证明）。

## 4. 信息架构（站点地图）

```
/                       首页：一句立意 + 情绪输入入口（核心 CTA）
/analyze                情绪结果 + 匹配 + 可视化页（分析后跳转）
/room/:slug             匿名情绪房间（WebSocket 群聊）
/timeline               个人情绪轨迹（历史情绪 + 离场总结回顾）
/about                  关于/玩法（可选）
```

### 关键页面与交互基调

- **首页**：极简，一句"先说说此刻的心情吧" + 大输入框；低饱和暗色 + 缓动粒子背景。
- **分析结果页**：情绪解读卡 + **情绪色盘可视化**（用户是发光点，6 间房是色块，连线到最近房）+ "进入房间"CTA；此页是"情绪→匹配闭环"的可视化证明。
- **房间页**：全屏房间色场 + 消息流（气泡带情绪色头像）+ 顶部房间主题/在线人数 + 底部输入；AI 主持发言样式柔和区分。
- **轨迹页**：时间轴 + 历史情绪标签 + 离场总结回顾（记忆回顾加分）。

## 5. 主场景 Happy Path（演示主线）

1. 小宇深夜失眠，打开 VibeChat，输入"明天要汇报，脑子停不下来，睡不着"。
2. AI 分析：主情绪=焦虑，valence=-0.7，arousal=0.8，intensity=0.85，解读"你好像被明天的汇报压着"。
3. 匹配：与『深夜焦虑』房 87% 同频 → 色盘上发光点连向该房色块。
4. 进房：系统给昵称"失眠的云" + 蓝紫色几何头像；房间已有 2 人 + 1 个兜底 bot。
5. AI 主持破冰："这间房的人都被明天的某件事压着，要不要说说你明天要面对什么？"
6. 小宇与同频的人匿名群聊，情绪被接住；冷场时 AI 抛话题。
7. 离场：AI 总结"你今晚的焦虑来自对汇报的不确定，但说出来好像轻了一点" → 存入轨迹。
8. （演示加分）现场切换 `LLM_PROVIDER`，OpenAI/Anthropic 各跑一遍同一输入，结果一致可演示。

## 6. MVP 边界（时间紧时保此即可拿主要分）

输入情绪 → LLM 情绪分析（结构化）→ 向量匹配进最近房 → WebSocket 匿名群聊（含兜底 bot）→ AI 破冰 + 离场总结 → 情绪可视化（色盘）→ 双接口切换可用 → 线上可访问 → 降级兜底。
