## 1. 兜底机制（匿名社交最怕"没人"，巡梦血的教训）

### 1.1 兜底 bot（房间永不空）

- 每间房常驻 1–2 个兜底 bot，`sender_role=bot`，昵称如"匿名居民""同频的影子"。
- **触发**：房间真人 < 阈值（如 0 或 1）或长时间无真人消息 → bot 以匿名身份发预设话术/回应。
- bot **不冒充真人**：视觉上与真人略有区分（或至少不刻意伪装），避免误导；但其发言服务于"房间有温度"。
- 话术：按 `room.slug` 配 `{greetings[], replies[]}`，结合关键词简单匹配回复（MVP 不调 LLM，快且零成本）。

### 1.2 无 key / LLM 失败降级矩阵

| AI 环节 | 正常 | 降级（无 key / 失败） |
|---|---|---|
| 情绪分析 | LLM 结构化输出 | **规则推断**：关键词→情绪向量（如"睡不着/明天"→焦虑，valence↓ arousal↑）；`provider=rule-fallback` |
| 匹配 | 向量 cosine | 不受影响（向量来自上一步，规则向量也可算） |
| AI 破冰 | LLM 生成 | 房间预设话术表 |
| 冷场抛话题 | LLM 生成 | 房间预设话题轮播 |
| 离场总结 | LLM 生成 | 模板总结 |
| 匿名群聊 | WebSocket | **不受影响**（聊天不依赖 LLM） |

> 设计意图：**断网 / key 失效也能演完整条"情绪输入→匹配→匿名对话"链路**。降级在 UI 明确标注（如"当前为离线模式"），避免评审误判（巡梦避坑）。

### 1.3 mock 多用户演示（赛题明确提示）

- **方案 A（最简）**：多浏览器窗口/无痕窗口，各自输入不同情绪文本 → 进同一房间 → 演示匹配与多人收发。
- **方案 B（脚本）**：后端调试接口 / Python 脚本，模拟若干 session 并发进房发消息（演示录制更稳）。
- 预置 2–3 个"演示情绪文本"卡片，一键填入（如焦虑/开心/emo各一），降低现场组织语言成本。

## 2. 部署：kang 服务器

### 2.1 部署架构

```
用户 ──HTTPS──► nginx (现有 SSL 证书, 域名 vibechat.wukangkang.com)
                   ├── /            ─► Next.js  (127.0.0.1:3000)
                   ├── /api/        ─► FastAPI  (127.0.0.1:8000)
                   └── /api/ws/     ─► FastAPI WS (需 Upgrade 头)
```

- **SSH 直连**（本机已有密钥）。
- **SSL**：复用服务器现有证书（与其他项目同）。
- **域名**：`vibechat.wukangkang.com`。
- **守护**：FastAPI（uvicorn）+ Next.js（npm start）各用服务器现有进程管理（systemd / pm2，按"其他项目一样"）。

### 2.2 nginx 配置要点（WebSocket 必须）

```nginx
server {
    listen 443 ssl http2;
    server_name vibechat.wukangkang.com;
    ssl_certificate     /path/to/existing.crt;   # 复用现有
    ssl_certificate_key /path/to/existing.key;

    # Next.js 前端
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    # REST API
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }
    # WebSocket（关键：Upgrade）
    location /api/ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;        # 长连接
    }
}
```

### 2.3 部署待确认清单（到服务器后核对）

> 用户说"和其他项目一样"，以下需现场对齐既有范式：

- [ ] OS、Node 版本（≥18）、Python 版本（≥3.11）
- [ ] nginx 配置目录与 reload 方式
- [ ] 现有 SSL 证书路径（`.crt` / `.key`）与有效期
- [ ] 进程守护方式（systemd / pm2 / supervisor）与既有服务范例
- [ ] 端口规划（3000 / 8000 是否被占）
- [ ] 其他项目的 nginx 反代范式（好照搬）
- [ ] 是否需防火墙放行 / 反代到 unix socket

### 2.4 部署步骤（骨架，待确认后细化）

1. SSH 上服务器，拉代码（或 rsync）。
2. 后端：`python -m venv .venv && pip install -r requirements.txt`，配 `.env`，`uvicorn app.main:app --host 127.0.0.1 --port 8000`（守护）。
3. 前端：`npm ci && npm run build && npm start`（守护，监听 3000）。
4. nginx 加 `vibechat.wukangkang.com` server 块（上配置），`nginx -t && nginx -s reload`。
5. 烟测：`curl https://vibechat.wukangkang.com/api/health`、浏览器开站点、WS 连接、双 provider 各跑一次。
