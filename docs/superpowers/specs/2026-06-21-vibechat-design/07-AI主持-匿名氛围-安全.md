# 07 · AI 主持 · 匿名氛围 · 安全

---

## 1. AI 房间主持工作流（确定性 workflow，不上 agent）

全部**事件驱动 + 单次 LLM 调用 + JSON schema 契约**（巡梦教训：路径清晰不上 agent）。主持发言 `sender_role=ai`，样式柔和区分，不喧宾夺主。

### 1.1 破冰（Icebreak）

| 项 | 内容 |
|---|---|
| 触发 | 新成员进房 / 房间创建后首条消息前 |
| 输入 | 房间 `vibe` + 新成员 `primary`/`reading` + 最近消息（可能空） |
| LLM 输出 | `{message}` 一句贴合氛围的开场/话题（≤40 字） |
| 降级 | 按 `room.slug` 取预设话术表（见 08） |

Prompt 要点：第二人称、温和、不评判、给一个低门槛的话题钩子。

### 1.2 冷场抛话题（Nudge）【P1：MVP 靠兜底 bot 顶，余量再做】

| 项 | 内容 |
|---|---|
| 触发 | 距上一条消息 > 阈值（默认 90s）且房间有人在线 |
| 输入 | 房间 `vibe` + 在线成员情绪聚合 + 最近 N 条消息 |
| LLM 输出 | `{message}` 温和抛话题 |
| 限频 | 同房间每 N 分钟最多 1 次，避免刷屏 |
| 降级 | 房间预设话题轮播 |

### 1.3 离场总结（Summary）

| 项 | 内容 |
|---|---|
| 触发 | 用户点"离开房间" |
| 输入 | 该用户本次房间的消息 + 进房情绪 + 房间主题 |
| LLM 输出 | `{summary, takeaway, mood_end}`（summary≤50 字回顾，takeaway≤20 字带走的话；**mood_end={valence,arousal} 复用于离场情绪旅程，见 [10 §6](./10-情绪场差异化设计.md)**） |
| 落库 | `emotion_timeline`（记忆回顾加分） |
| 降级 | 模板"你在『XX』房聊了 N 条，情绪似乎从 {primary} 开始" |

### Prompt 契约（示意）

```
SYSTEM: 你是匿名情绪房间的温和主持。用第二人称、克制、不评判的语气，
只输出符合 schema 的 JSON。
USER: 房间氛围：{vibe}；成员情绪：{primary}-{reading}；最近对话：{recent}
输出：{"message": "..."}
```

## 2. 匿名身份生成（系统生成，绝不暴露真实身份）

### 2.1 匿名昵称

- **MVP（推荐，快且稳）**：每间房配词库 `{adjectives:[], nouns:[]}`，按 `room.slug` 随机组合 `形容词+名词`，序号去重。
  - 例（深夜焦虑）：形容词 `失眠的/停不下的/清醒的/辗转的`，名词 `云/钟/灯/信号` → "失眠的云""停不下的钟"。
- **P1（可选）**：LLM 生成贴合房间情绪的昵称（调 LLM，有延迟/成本，演示期不优先）。

### 2.2 情绪色几何头像

- **纯 SVG/CSS 生成，零依赖**。
- 算法：
  - 颜色：由用户 `valence`(色相) × `arousal`(明度) 映射 HSL → 一个情绪色。
  - 形状：`hash(session_id + room_slug) % N` 选圆/三角/六边形/方块。
  - 渲染：情绪色渐变几何块 + 细描边。
- 同一用户在同一房内头像稳定（进房快照存 `room_memberships.avatar`）。

## 3. 视觉氛围规范（体验 20% 载体）

| 元素 | 规范 |
|---|---|
| 全局底色 | 暗色低饱和（如 `#0E0F1A`），大量留白 |
| 房间色场 | 房间主色径向渐变背景 + 轻量漂浮粒子（CSS/Canvas，移动端降级为静态渐变） |
| 情绪色盘 | 见 [03 §7](./03-情绪匹配引擎.md)：用户发光点 + 房间色块 + 同频连线 |
| 动效 | 呼吸（opacity 缓动）、消息浮入、进房淡入；慢而柔；**避免重型 3D** |
| 文案 tone | 第二人称、温柔克制、不评判：如"先说说此刻的心情吧""这间房的人都在" |
| 字体 | 无衬线、字重适中、行高舒展 |

文案库（统一管理按钮/空状态/错误/降级提示），避免散落硬编码。

## 4. 安全边界（赛题点名项）

### 4.1 极端情绪 / 自伤检测（双保险）

- **LLM 侧**：情绪分析输出 `risk_flag`（见 03），命中极端/自伤倾向置 true。
- **关键词侧**：正则匹配自伤/绝望/轻生类词（词表可配）。
- 任一命中 → 触发**求助卡片**：

  > 「如果你现在很难受，你不需要独自扛。这里有人可以帮你。」
  > - 全国心理援助热线（希望24）：400-161-9995
  > - 北京心理危机研究与干预中心：010-82951332
  >
  > （号码请上线前核实官方最新公布）

- 求助卡片**柔和、不阻断**（可关闭），不以弹窗惊吓用户。

### 4.2 匿名保护

- 不采集/不展示真实身份（无手机号/邮箱/真实昵称）。
- 昵称头像系统生成，会话内稳定、跨会话可变。

### 4.3 输入校验与防刷（巡梦安全三件套）

- 输入校验：非空、长度上限（如 ≤500 字）、基础清洗。
- 配额：每 session 每日情绪分析上限（如游客 10 / 登录 60）、消息速率限制（防刷屏）。
- 会话隔离：所有数据按 `session_id` 隔离，互不可见。

## 5. 可测试性

- 主持三工作流的触发条件、限频、降级路由为纯逻辑，单测覆盖。
- 昵称生成、头像色相映射、`risk_flag` + 关键词双保险为纯函数，单测覆盖。
- 求助卡片触发链路（risk→卡片）单测 + 手动验收。
