# 06 · 数据模型

---

## 1. 约定

- **主键统一 UUID 字符串**，禁止自增（全局规范，利于分布式/合并）。
- 时间统一 **UTC**，存 `TIMESTAMPTZ`。
- 灵活结构（向量/标签/头像）用 **JSON 列**内联（演示期降低复杂度）。
- 软删可选；演示期直接物理删或加 `deleted_at`。
- 引擎 SQLite（默认，零依赖），DDL 兼容 PostgreSQL（可平滑切换）。

## 2. 表结构

### 2.1 `sessions`（匿名会话）

### 2.5 `messages`（房间消息流）

## 3. 关系

```
sessions 1───N emotion_analyses
sessions 1───N room_memberships N───1 rooms
sessions 1───N messages         N───1 rooms
sessions 1───N emotion_timeline
emotion_timeline N───1 emotion_analyses (可选)
```

---

## 4. 枚举

- `sender_role`: `user` | `ai` | `bot`
- `provider`: `openai` | `anthropic` | `rule-fallback`
- `primary` 标签集：见 [03 §4](./03-情绪匹配引擎.md)

---

## 5. 关键索引

| 索引 | 用途 |
|---|---|
| `messages(room_id, created_at)` | 房间消息流按时间拉取 |
| `emotion_analyses(session_id, created_at)` | 用户情绪历史 |
| `room_memberships(room_id) where left_at IS NULL` | 房间当前在线 |
| `emotion_timeline(session_id, created_at)` | 轨迹页 |
| `rooms.slug` UNIQUE | 按 slug 查房 |

---

## 6. 初始化（seed）

- `backend/app/seeds/rooms.seed.json`：内含 6 间房的 `slug/name/description/anchor_vector/color/vibe`（数值见 [03 §5](./03-情绪匹配引擎.md)）。
- 启动时若 `rooms` 表为空，自动从 seed 载入（`create_all` + seed）。
- 房间 anchor 向量自此固定缓存，匹配时直接读，不重复调 LLM。

## 7. 模型示意（SQLAlchemy，节选）

```python
class Message(Base):
    __tablename__ = "messages"
    id: Mapped[str] = mapped_column(String, primary_key=True, default=uuid4_str)
    room_id: Mapped[str] = mapped_column(ForeignKey("rooms.id"), index=True)
    session_id: Mapped[Optional[str]] = mapped_column(ForeignKey("sessions.id"), nullable=True)
    sender_role: Mapped[str]                        # user|ai|bot
    nickname: Mapped[str]
    content: Mapped[str]
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now(), index=True)
```

> 完整模型见实现；本表为字段/关系/索引的契约，验收时逐字段核对。
