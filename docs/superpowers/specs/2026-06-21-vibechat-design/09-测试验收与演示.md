# 09 · 测试验收与演示

> 按选手要求，**测试验收在实现完成后进行**（节省前期时间）。本文先备好清单与脚本，供实现收尾时直接用。

---

## 1. 测试策略

- **核心纯函数单测（必做）**：向量构造、加权 cosine、同频度、匹配排序/平局、双接口响应归一化、工厂路由、昵称/头像生成、risk 检测、主持触发/限频。
- **适配层测试（不调真 API）**：`respx`/fake transport 拦截，断言两 client 的请求体构造与响应解析。
- **集成/手动验收**：端到端跑核心链路（对应 §3 清单）。
- 框架：后端 `pytest`；前端关键逻辑可用 `vitest`（视余量）。

> 巡梦避坑：至少给核心纯函数补单测；不追求全覆盖，追求"最易错且最该稳"的部分有保护。

---

## 2. 单测清单（后端 `pytest`）

| 模块 | 用例 |
|---|---|
| `matching` | 向量构造、加权 cosine 正确性、同频度映射、argmax 选房、平局取 arousal 近者 |
| `llm/openai_client` | 请求体含 `response_format`、解析 `choices[0].message.content`/usage |
| `llm/anthropic_client` | `system` 单独传、prefill `{"`、解析 `content[].text`/usage |
| `llm/factory` | `LLM_PROVIDER` 取值 → 对应实现 |
| `llm/normalize` | 两路响应 → 统一 `LLMResponse`；结构化 `raw_json` 解析（含宽容解析） |
| `host_service` | 破冰/冷场/总结触发条件、冷场限频、降级路由 |
| `emotion`（规则降级） | 关键词→向量推断 |
| `identity` | 昵称组合去重、头像色相映射稳定性 |
| `safety` | `risk_flag` + 关键词双保险 → 求助卡片触发 |

## 3. 验收清单（逐条对齐赛题必做 + 4 个严重扣分项）

### 必做 7 项

- [ ] 情绪输入 + LLM 分析（标签/向量/强度/正负向/关键词/解读）
- [ ] 情绪匹配（向量 cosine → 进最近房）+ 同频度展示
- [ ] 未匹配/无真人兜底（兜底 bot）
- [ ] 匿名文本对话（WebSocket 收发、时间序、在线状态）
- [ ] 匿名身份系统生成（昵称 + 情绪色头像）
- [ ] 双接口适配（OpenAI + Anthropic，env 切换，README 说明）
- [ ] 前后端完整（FastAPI + Next.js）
- [ ] 线上演示地址可访问（`vibechat.wukangkang.com`）
- [ ] 稳定演示（LLM 异常降级、非静态可运行）

### 4 个严重扣分项反向确认

- [ ] ① 线上地址可访问（赛前烟测通过）
- [ ] ② 双标准都真实跑通（现场切 provider 各跑一遍 + 单测）
- [ ] ③ 情绪真正影响匹配（同频度由向量算出并展示 + `emotion_analyses` 留痕）
- [ ] ④ 匿名聊天真实收发（mock 多用户多端验证）

## 5. 100 字产品介绍（提交材料定稿）

> VibeChat 是 AI 驱动的情绪社交应用。输入此刻心情，AI 解析情绪色彩与强度，把同频的人匹配进同一间匿名房间，由 AI 主持破冰、引导对话、离场给出情绪小结。不是给聊天室贴标签，而是让"当下情绪"真正决定你和谁相遇。

（约 95 字，符合 ≤100 字）
