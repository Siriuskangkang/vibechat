## 1. 一句话定位

**VibeChat 是一款 AI 驱动的情绪社交应用：输入此刻心情，AI 解析其中的情绪色彩与强度，把同频的人匹配进同一间匿名房间，由 AI 主持破冰、引导对话、离场给出情绪小结。不是给聊天室贴情绪标签，而是让"当下情绪"真正决定你和谁相遇。**

## 2. 产品魂（贯穿全局）

> **「同一秒，被同一种心情抓住的人，值得在一间房里相遇。」**

绝大多数线上社交靠**身份 / 兴趣 / 位置 / 关系链**匹配，但人真正想说话时，先要被**当下情绪**理解。VibeChat 只做一件事：

1. 你丢进一段心情 → AI 听懂它的色彩；
2. 把你送进一间此刻和你**情绪同频**的匿名房间；
3. AI 做主持、匿名做保护，让"说不清的情绪"变成"一次被理解的相遇"。

三条贯穿全局的设计原则（沿用第二期巡梦的方法论）：

- **情绪闭环 > 情绪标签**：情绪分析必须真正驱动匹配，可解释、可量化、可视化（这是赛题的灵魂与最大扣分点）。
- **氛围即产品**：界面、文案、动效、色场全部服务于"情绪社交"主题，不是"普通聊天室 + AI 标签"。
- **温柔即正确**：文案克制、节奏柔和、安全兜底，不打扰、不评判；极端情绪有求助出口。

> **★ 产品最大特点（题目之外的核心竞争力）**：把情绪做成「流动的场」——**会呼吸的情绪房间**。房间色调、共鸣度、离场旅程随同处一室者的情绪实时流动，彻底区别于"贴情绪标签的聊天室"。详见 [10-情绪场差异化设计](./10-情绪场差异化设计.md)。

## 3. 技术栈一览

| 层 | 选型 | 理由 |
|---|---|---|
| 前端 | **Next.js（App Router）+ React + TypeScript + Tailwind CSS** | 赛题指定 Next.js；TS 提升工程质量分 |
| 实时通信 | **WebSocket**（前端原生 + 后端 FastAPI） | 房间群聊需双向实时（发消息、在线状态、AI 主持推送） |
| 后端 | **Python FastAPI（async）** | 赛题指定；异步并发调 LLM 合适；Pydantic 类型契约 |
| ORM / DB | **SQLAlchemy 2.0 + SQLite（默认）** | 零外部依赖，利于"演示不崩"（巡梦教训）；可平滑切 PostgreSQL |
| 配置 | **pydantic-settings** 读 `.env` | 密钥不入库，符合安全规范 |
| LLM 适配 | **统一抽象层**，双实现：OpenAI 标准（百炼 / Qwen）+ Anthropic 标准（DeepSeek `/anthropic`） | 赛题硬性双标准接口；环境变量一键切换 |
| 部署 | kang 服务器，nginx 反代 + 复用现有 SSL，域名 `vibechat.wukangkang.com` | 与其他项目同范式 |

## 3.5 🎯 MVP 范围与轻量化原则

> 遵循第二期巡梦经验沉淀（`docs/espoirdream-main/经验沉淀.md`）：**少依赖 = 少踩坑 = 稳定演示；高完成度 > 技术栈复杂度。能用一个依赖解决就别引三个。**

### 轻量化红线（全项目遵守）

- **前端零组件库**（不引 shadcn / antd / MUI 等）、**零重型图表库**；样式用 Tailwind 或纯 CSS；可视化**纯 SVG / Canvas / CSS**。
- **后端 SQLite 零配置**、最小依赖；FastAPI 单入口。
- **代码量目标 ~4k 行**（对标巡梦：后端 ~1.7k + 前端 ~2.3k，功能齐全却不臃肿）。

### P0（MVP 必做——保此即可拿主要分）

1. 情绪输入 → LLM 结构化分析（VAD 向量 + 标签 + 解读）
2. 向量匹配进最近房 + 同频度展示 + **情绪色盘可视化**
3. WebSocket 匿名房间群聊（含兜底 bot）
4. 匿名身份系统生成（昵称 + 情绪色几何头像）
5. AI 破冰 + 离场总结（展示）
6. 双接口适配（OpenAI + Anthropic，env 切换）
7. 线上部署（`vibechat.wukangkang.com`）+ 无 key 降级

### P1（余量再加，不阻塞 MVP）

- 冷场抛话题（MVP 靠兜底 bot 顶）
- `emotion_timeline` 建表 + 轨迹页（MVP 离场总结先展示不落库）
- 严格配额防刷（MVP 宽松）

### 砍掉（YAGNI，明确不做）

- 情绪雷达图（色盘已证明闭环）
- 互补匹配模式（专注相似匹配）
- embedding 扩展点（只用 VAD）
- 流式输出（同步即可）
- Anthropic `tool_use` 增强（只用 prefill）
