# 03 · 情绪匹配引擎

> 本项目的灵魂。所有"情绪是否真正影响匹配"的质疑，都在这里用可量化、可解释、可可视化的方式回应。

---

## 1. 设计目标

- 情绪分析结果必须**实打实参与匹配**（扣分项③反向证明）。
- 匹配依据**可解释**（用户看得懂为什么进这间房）。
- 匹配结果**可量化展示**（"同频度 87%"）。
- 匹配过程**可可视化**（情绪色盘落点）。

---

## 2. 为什么用 VAD 维度向量，而非 embedding API

| 方案 | 优点 | 缺点 | 结论 |
|---|---|---|---|
| **LLM 输出 VAD 维度向量（选定）** | 可解释、契合赛题字段(强度/正负向)、天然可视化、不依赖额外 API | 维度有限（4维） | ✅ |
| embedding API（把情绪文本转向量） | 高维、对自由文本鲁棒 | DeepSeek `/anthropic` 无 embedding；百炼需额外调；**不可解释**（512维用户看不懂）；可视化要降维 | ❌ |

**选定理由**：DeepSeek 的 Anthropic 兼容端点不提供 embedding；依赖 embedding 会卡住一路。而 VAD（Valence 正负 / Arousal 唤醒 / Intensity 强度 / Social 社交倾向）既满足赛题"建议含强度、正负向"，又直接可解释、可画图。

> 不引入 embedding：本设计只用 VAD 维度向量，简化依赖与实现（YAGNI，不再保留 embedding 扩展口子）。

---

## 3. 情绪向量模型（LLM 单次结构化输出）

```json
{
  "primary": "焦虑",              // 主情绪标签（从 §4 标签集选）
  "secondary": ["失眠","过度思考"], // 次要标签（0-3 个）
  "valence": -0.7,                // 正负向 [-1 消极, 1 积极]
  "arousal": 0.8,                 // 唤醒/激动度 [0 平静, 1 激动]
  "intensity": 0.85,              // 强度 [0,1]
  "social": 0.6,                  // 社交倾向 [0 想被陪伴, 1 想倾诉]
  "keywords": ["明天","汇报","睡不着"],
  "reading": "你好像被明天的汇报压着，脑子停不下来。", // 一句解读（前端展示）
  "risk_flag": false              // 极端/自伤风险（安全边界用，见 07）
}
```

**用于匹配的向量** = `[valence, arousal, intensity, social]`（4 维）。

### 字段含义与取值约束（写进 prompt）

- `valence`：情绪正负，-1(很消极) ~ 1(很积极)，0 为中性。
- `arousal`：生理/心理激活度，0(平静/低落迟钝) ~ 1(激动/紧张亢奋)。
- `intensity`：整体情绪强度，0(微弱) ~ 1(极强)。
- `social`：互动倾向，0(想静静被陪伴) ~ 1(想倾诉表达)。

---

## 4. 情绪标签集（primary 候选）

`焦虑 / 抑郁 / 愤怒 / 喜悦 / 平静 / 迷茫 / 孤独 / 疲惫 / 兴奋 / 感动 / 失落 / 紧张 / 委屈 / 释然`

> 标签用于**展示与解读**，匹配**以向量为准**（不靠标签硬分类，避免被评成"只是分类"）。

## 5. 预设房间清单（6 间，anchor 向量为人工标定 seed）

| 房间 | slug | anchor 向量 [val,arousal,intensity,social] | 主色 | 氛围 | 典型破冰示例 |
|---|---|---|---|---|---|
| 深夜焦虑 | `late-night-anxiety` | `[-0.6, 0.80, 0.80, 0.60]` | 蓝紫 `#6D5BFF` | 紧绷、低频嗡鸣 | "这间房的人都被明天的某件事压着，说说你明天要面对什么？" |
| emo 时刻 | `emo` | `[-0.50, 0.30, 0.60, 0.55]` | 灰蓝 `#7C8DB5` | 低沉、缓慢 | "好像都有点闷，不用急着说话，待着也行。" |
| 开心想分享 | `joyful-share` | `[+0.70, 0.60, 0.60, 0.75]` | 暖金粉 `#FFB26B` | 明亮、上扬 | "有开心事就抖出来吧，这间房专收好消息。" |
| 愤怒吐槽 | `vent-anger` | `[-0.60, 0.85, 0.85, 0.70]` | 红橙 `#FF6B5E` | 炽烈、张力 | "被什么气到了？先骂为敬，我们听着。" |
| 平静独处 | `calm-solo` | `[+0.30, 0.15, 0.25, 0.20]` | 青绿 `#5BCFA8` | 柔和、留白 | "今天挺安静的，想随便聊聊还是就想待会儿？" |
| 迷茫求建议 | `lost-advice` | `[-0.20, 0.40, 0.50, 0.80]` | 紫 `#9B7BE0` | 思忖、低张力 | "卡住了？说说你正纠结的事，也许大家能帮着捋一捋。" |

### 锚定向量如何确定

- **MVP：人工标定 seed**（上表数值，稳定/可复现/可解释，写进 `rooms.seed` 配置）。
- 可选：用 LLM 对每间房 `description` 跑一次"该房间典型访客的情绪向量"离线生成，覆盖 seed（README 注明可切换）。
- 房间向量缓存于 `rooms.anchor_vector`（JSON 列），运行时直接读，不重复调 LLM。

## 6. 相似度算法

```
用户向量 U = [valence_u, arousal_u, intensity_u, social_u]
房间向量 R = [valence_r, arousal_r, intensity_r, social_r]
权重      W = [1.0,        1.0,        0.6,         0.5]      # 正负/唤醒为主

加权 cosine 相似度：
   sim(U,R) = (U·(W*R)) / (||U|| * ||W*R||)

同频度 = round( max(0, sim) × 100 )     # 百分比，避免负值

匹配 = argmax over rooms( sim(U, R) )    # 取同频度最高的房
```

- **权重理由**：valence（正负）与 arousal（激动度）最能区分"情绪状态"，权重最高；intensity、social 为辅助维度。
- **平局处理**（两房 sim 差 < 0.02）：MVP 取 arousal 更接近者（更"同频"）；P1 可并列展示让用户选。
- 同频度展示：分析结果页 + 房间页顶部都显示"你与此房 X% 同频"。

### 匹配流程（`services/matching.py`）

1. 取用户向量 U（来自情绪分析）。
2. 遍历 6 间房 R，算 sim。
3. 排序 → 最高即匹配房；同时返回 top-3 供"也可看看"。
4. 进房即 `INSERT room_memberships`，WS 加入房间 channel。

## 7. 情绪可视化（赛题加分项落地）

### 7.1 情绪色盘（核心可视化）

- 二维平面：**x = valence（-1..1，左消极右积极）**，**y = arousal（0..1，下平静上激动）**。
- 用户是一个**发光点**，位置 = (valence, arousal)，颜色 = 由 valence 决定色相、arousal 决定明度。
- 6 间房是平面上的**色块**（按各自 anchor 的 valence/arousal 定位，颜色 = 房间主色）。
- **连线**：用户点 → 匹配房间块，标注"87% 同频"。
- 这张图直接把"为什么进这间房"画给评审看。

### 7.2 ~~情绪雷达图~~（已砍：YAGNI）

> 色盘已充分证明匹配闭环，雷达图不做，省一个可视化、少一个依赖。

### 7.3 房间色场

房间页背景 = 房间主色的粒子/渐变 + 缓动，消息气泡头像用情绪色（P0 用纯 CSS/Canvas，避免重型库）。

## 8. 情绪分析 LLM Prompt 契约（示意）

```
SYSTEM:
你是情绪分析专家。读取用户的一段心情文字，输出严格符合下方 JSON schema 的结果，
不要输出任何 JSON 以外的内容。数值字段严格落在指定区间。

USER:
文本："{用户输入}"

要求输出 JSON：
{primary, secondary[], valence∈[-1,1], arousal∈[0,1], intensity∈[0,1],
 social∈[0,1], keywords[], reading(<=30字), risk_flag(bool)}
```

- **few-shot**：补 2 个示例（一个焦虑高 arousal、一个平静低 arousal），稳定数值输出（巡梦避坑）。
- **结构化获取**：OpenAI 走 `response_format=json_object`；Anthropic 走 `tool_use` 或 prefill `"{\""`（见 [05](./05-双接口适配层.md)）。

## 9. 边界与异常

| 边界 | 策略 |
|---|---|
| 空输入 | 前端禁用提交；后端 400 `"先说说此刻的心情吧"` |
| 超长输入 | 前端限制（如 ≤500 字）+ 后端截断 |
| LLM 返回非法 JSON | 重试 1 次 → 仍失败 → **降级**：关键词规则推断 valence/arousal（见 [08](./08-兜底与部署.md)），标记 `provider=rule-fallback` |
| 所有房间同频度都极低（<30%） | 仍进最高者；P1 可提示"此刻比较独特，先去 XX 房看看" |
| 极端情绪（risk_flag=true） | 进房前/中触发求助卡片（见 [07](./07-AI主持-匿名氛围-安全.md)） |

---

## 10. 可测试性

- 向量构造、加权 cosine、同频度计算、平局处理均为**纯函数**，单独单测（覆盖正/负 valence、平局、零向量）。
- 情绪分析结果落库 `emotion_analyses`（含 provider/model），验收可回溯"这次匹配基于哪个向量"。
